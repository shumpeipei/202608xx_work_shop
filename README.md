# 保守性重視の開発実践 勉強会

SE保守・運用職からSE開発職への転向希望者を対象にした、リファクタリングと保守性の高いコード設計を学ぶワークショップ資料です。

## 概要

| 項目     | 内容                          |
| -------- | ----------------------------- |
| 対象     | SE開発転向希望者              |
| 形式     | オンラインワークショップ      |
| 使用言語 | Python / Java（どちらか選択） |

## 学習目標

- DRY原則・SOLID原則の基礎を理解する
- コード臭（Code Smell）を発見し、リファクタリングで改善できる
- 仕様変更が発生した際の変更影響範囲を分析できる
- 「動けばいい」から「変えやすく・読みやすく・テストしやすい」への思考転換

## 事前準備

演習コードを手元で実行する場合は、以下のいずれかをインストールしてください。

### Python を使う場合

**必要なバージョン**: Python 3.10 以上

```bash
# バージョン確認
python3 --version
```

インストールされていない場合は [python.org](https://www.python.org/downloads/) からダウンロードしてください。

### Java を使う場合

**必要なバージョン**: JDK 17 以上（`record` 構文を使用）

```bash
# バージョン確認
java -version
javac -version
```

インストールされていない場合は [Adoptium](https://adoptium.net/) から Temurin JDK をダウンロードしてください。

## 演習の概要

`02_演習_問題コード` には、意図的に以下の問題点を含んだ注文管理システムのコードが入っています。

- 同じ顧客取得ロジックが3箇所に重複している（DRY違反）
- 1つの関数が計算・メール送信・在庫更新を一手に担っている（SRP違反）
- 意味不明な1文字変数（`c`, `t`, `s` など）
- マジックナンバーの直書き（`3000`, `5000`, `0.9` など）
- エラー処理なし（存在しない顧客IDが渡された場合）

これらを `03_演習_模範解答` と見比べることで、リファクタリングの効果を体感できます。

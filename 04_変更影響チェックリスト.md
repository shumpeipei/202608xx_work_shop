## 【参考】問題コード vs 模範解答コードの比較

### 問題コード（リファクタリング前）での変更コスト

**割引コード `NEW30` 追加**
```python
# process_order 関数内の if/elif に1行追加（この関数だけ変えればいい? → 他に同様のロジックがあれば全箇所！）
elif dc == "NEW30":
    t = t * 0.7
```
- 修正箇所数: `process_order` の中の `if/elif` ブロック（1箇所）
- ただし、将来 `process_return` などでも同じ割引を適用したい場合、そちらにも追記が必要

**送料無料基準を 8,000円 に変更**
```python
# マジックナンバーを探して修正（5000 がどこに何個あるか grep が必要）
elif t >= 5000:   # ← ここを 8000 に
    s = 0
```

### 模範解答コード（リファクタリング後）での変更コスト

**割引コード `NEW30` 追加**
```python
# 定数辞書に1行追加するだけ
DISCOUNT_RATES = {
    "GOLD10":   0.10,
    "SILVER5":  0.05,
    "SUMMER20": 0.20,
    "WINTER15": 0.15,
    "NEW30":    0.30,  # ← これだけ
}
```

**送料無料基準を 8,000円 に変更**
```python
# 定数テーブルの数値を1箇所変更するだけ
SHIPPING_RULES = [
    (8000, 0),    # ← 5000 → 8000 に変更（ここだけ）
    (3000, 300),
    (0,    500),
]
```

> **気づき**: リファクタリングによってコードの「変更容易性」が大幅に向上し、  
> 修正箇所が減り、修正漏れによるバグのリスクも下がることを確認しましょう。
